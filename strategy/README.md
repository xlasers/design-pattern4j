##### 目的

封装好一组不同逻辑的方法,各个方法间可以互相交换;策略模式在独立于客户端下,可以使用不同的策略


##### 适用性

* 同一个类:某种情况下造成if,else条件语句分支过多,可以用策略模式改造,增加可读性且便于维护.
* 多个类:多个相关类如果仅仅是方法行为不同,可以使用策略模式.
